# Logistics-Optimization-for-Delivery-Routes-Flipkart

# Overview
Designed an analytics system utilizing SQL to optimize Flipkart delivery routes and boost shipment efficiency. Performed extensive data cleaning, managing duplicates, null entries, and unifying date formats. Applied SQL methodologies, including aggregations and window functions, to evaluate logistic processes and improve delivery performances.

# Tools Used
SQL

# Key Insights
- Multiple routes have more than 20% delayed shipments
- Several warehouses operate above the global average processing time
- Some delivery agents have on-time performance below 80%
- Orders with multiple delayed checkpoints were identified, indicating critical delivery failures
These findings confirm that inefficiencies are concentrated and measurable, making them actionable.

# Conclusion
The analysis reveals that delivery delays are primarily influenced by three key factors:
- Route inefficiencies, where certain routes exhibit a high percentage of delayed shipments, indicating consistent performance issues.
- Warehouse bottlenecks, where some warehouses operate above the global average processing time, contributing to slower order dispatch.
- Delivery agent performance variability, where a subset of agents shows lower on-time delivery rates, impacting overall delivery success.
Further investigation using shipment tracking data highlights that delays are often linked to identifiable operational factors, and some orders experience multiple delayed checkpoints, indicating deeper systemic issues.
By combining route-level metrics, warehouse performance, and agent analysis, this project provides a comprehensive view of logistics operations and highlights areas that require optimization.
Overall, the findings suggest that improving route planning, optimizing warehouse processing workflows, and enhancing agent performance can significantly improve delivery efficiency and on-time delivery rates.
